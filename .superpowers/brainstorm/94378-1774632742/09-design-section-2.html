DeepSeek AI 专家设计
  • 后端新增正式 AI service，不再用占位 explain
  • 统一接入 DEEPSEEK_API_KEY 和 DEEPSEEK_API_URL
  • 右键选中代码时，传递：选中文本 + 当前完整代码 + 当前章节上下文
  • AI 面板支持 4 类动作：解释、查错、提示、延伸练习
  • AI 回复进入持续型专家会话，而不是一次性弹窗
Python 工作台设计
  • 只支持 Python，明确写成产品范围，不做伪多语言
  • 编译器支持从 0 开始空白模板
  • 同时提供：课程示例模板、练习模板、框架模板
  • 运行后优先展示真实变量态和 stdout / stderr
  • 用户离开页面时自动保存草稿与最近位置
内容补全策略（明确纳入首版）

你刚才强调的“后续一定要把内容补上”，这里不作为备注，而是直接写进首版产品定义：

  • 每个已上线章节必须同时包含：知识解释、示例代码、从 0 开始模板、至少 2 个习题、至少 1 个框架练习
  • 每个章节必须有可视化帧，不能只有代码编辑器
  • AI 专家必须知道当前章节目标，不能脱离上下文瞎回答
  • 第一版至少先把 Python 主路径做成完整可学，而不是只有 1-2 个 demo unit
  • 未完成内容不上线，不允许用“占位章节”假装课程完整

为什么这部分重要

  • 它决定这个产品是不是“真能用”
  • 它把 AI、编译器、课程内容真正绑到一起
  • 它避免做出只有漂亮界面的空壳

由此带来的约束

  • 首版范围要聚焦 Python 主路径
  • 不能一开始铺太多语言
  • 内容生产本身要进入实施计划，不是附属工作
