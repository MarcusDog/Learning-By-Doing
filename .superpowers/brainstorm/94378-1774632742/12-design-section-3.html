设计方案 · 第 3 部分

首版范围和明确延后项。目标是保证第一版既完整，又不会因为过度扩张而做成半成品。

首版必须完成
  • 多用户：邮箱 + 密码登录 / 注册 / 会话管理
  • App-First：登录后直接恢复上次学习位置
  • 高级应用壳层：Premium Hybrid 风格 + 三栏工作台
  • Monaco 编辑器 + Python runner
  • 右键学习动作系统
  • DeepSeek 持续型 AI 专家会话
  • 草稿自动保存、最近位置恢复、最近会话恢复
  • 至少一条完整 Python 主路径，内容补齐到可学状态
首版明确延后
  • 多语言编译器（JavaScript / C++ / Rust 等）
  • 内容后台 / 编辑后台
  • 多人协作编辑
  • 浏览器内 Python 运行作为正式主路径
  • 移动端完整编辑器体验优化到桌面同级
  • 订阅 / 支付 / 商业化系统
首版完成定义

“完成”不是指页面看起来像，而是用户能从注册开始，真正完成一次学习循环：

  1. 注册 / 登录
  2. 进入上次学习位置
  3. 打开一个 Python 章节
  4. 切换知识点 / 查看可视化
  5. 从示例、框架模板或空白模板开始写代码
  6. 运行代码并看到变量态和输出
  7. 右键选中代码，调用 AI 专家解释 / 查错 / 生成练习
  8. 保存学习进度，退出后再次回来还能恢复现场

这样收范围的好处

  • 首版就能形成完整学习闭环
  • 不会因为后台、多语言、支付把主线拖死
  • 最符合你当前“先做真正可用产品”的目标

这意味着什么

  • Python 内容必须做深，而不是做广
  • 首版必须接受“平台深度优先，平台广度后置”
  • 所有看起来高级但不影响学习闭环的功能都要排后
