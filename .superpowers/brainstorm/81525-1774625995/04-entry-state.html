打开网站后的第一屏

现在已经确定：整站是 App-First，主界面是三栏工作台。只决定用户一打开网站时，默认落到哪种状态。

A

直接回到上次学习位置

用户打开网站后，直接进入上次正在学的 unit studio。最像真正的生产力软件。

优点

  • 最高效，适合你自己天天用
  • 打开就能继续写代码
  • 软件感最强

代价

  • 第一次进入的人可能不清楚整体课程结构
B

先到课程工作台首页

先看到路径、最近进度、推荐继续学习的章节；再点进具体 studio。是更平衡的产品入口。

优点

  • 课程结构清楚
  • 仍然保持应用感
  • 适合以后多语言、多路径扩展

代价

  • 多一步点击才能进编译器
C

固定从 Python 起始单元开始

每次都默认落到 Python 第一课或一个固定起点。最简单，但更像 demo，不像成熟产品。

优点

  • 实现最简单
  • 首次体验最统一

代价

  • 不适合长期使用
  • 不能体现真实学习进度
我的推荐

推荐 A · 直接回到上次学习位置

因为你明确说这个站点首先是给你自己真正用的，不是先做展示型首页。A 最符合“打开就继续学”的真实产品体验。

如果你后面要公开给别人用，我会再补一个“工作台首页”作为二级入口，不会影响主路径。
